# Data Quality Validation Framework for Synthetic Housing Data

## Overview
This project demonstrates how raw operational data can be validated, cleaned, and prepared for reliable dashboard reporting.

## Business Problem
Dashboards and leadership reports are only useful when the underlying data is accurate, complete, and standardized.

## Dataset
The data is fully synthetic and safe for public use. It was intentionally created with common data quality issues.

## Tools Demonstrated
- SQL validation checks
- Data quality rules
- Before-and-after cleaning summary
- Data documentation
- Reporting readiness review

## Issues Simulated
- Missing building numbers
- Duplicate record IDs
- Invalid occupancy values
- Incorrect calculated percentages
- Invalid category codes
- Inconsistent installation naming

## Outcome
The cleaned file removes or corrects quality issues so the data can support repeatable reporting.
