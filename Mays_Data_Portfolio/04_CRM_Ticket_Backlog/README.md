# CRM Service Desk Performance & Backlog Dashboard

## Overview
This project uses synthetic CRM ticket data to analyze backlog, SLA performance, ticket categories, priority levels, and resolution trends.

## Business Problem
Operational leaders need visibility into ticket volume, aging backlog, SLA compliance, and support team performance.

## Dataset
The dataset is fully synthetic and does not contain real customer, employee, or agency data.

## Suggested Dashboard Visuals
1. KPI cards: Open Tickets, Closed Tickets, Average Days Open, SLA Compliance %
2. Bar chart: Tickets by Category
3. Line chart: Ticket Volume by Month
4. Stacked bar chart: SLA Met vs Missed by Team
5. Table: Aging Open Tickets
6. Filter: Priority, Status, Assigned Team, Category

## Key Findings to Highlight
1. Account Access and Report Request tickets often drive high volume.
2. Open-ticket aging creates risk when tickets exceed SLA targets.
3. SLA misses vary by team and priority.
4. Ticket backlog can be reduced with recurring review and escalation rules.

## Recommendations
1. Create weekly backlog review meetings.
2. Add aging-ticket alerts.
3. Improve knowledge base content for common ticket types.
4. Reassign older tickets before SLA breach.
