# Portfolio Build Guide

## Step 1: Upload to GitHub
Create a new repository called:

`anthony-mays-data-portfolio`

Upload this full folder. Make sure the README file appears on the main repository page.

## Step 2: Build the Army Housing Dashboard
Use:

`01_Army_Housing_Utilization/data/cleaned_synthetic_army_housing_data.csv`

Suggested dashboard:
- KPI cards: Total Beds, Occupied Beds, Vacant Beds, Average Net Use %
- Bar chart: Net Use % by Installation
- Line chart: Net Use % by Month
- Table: Buildings Below 70% Utilization
- Filter: Installation, Region, Housing Type, Month, Priority Level

## Step 3: Build the Data Quality Project Page
Use:
- `02_Data_Quality_ETL_Validation/data/messy_housing_data.csv`
- `02_Data_Quality_ETL_Validation/data/cleaned_housing_data.csv`
- `02_Data_Quality_ETL_Validation/data/data_quality_summary.csv`
- `02_Data_Quality_ETL_Validation/sql/01_data_quality_checks.sql`

This project should show that you can find, document, and correct bad data.

## Step 4: Build the HR Dashboard
Use:

`03_HR_Hiring_Pipeline/data/synthetic_hr_hiring_pipeline.csv`

Suggested dashboard:
- Hiring funnel
- Average time-to-fill by office
- Requisitions over 90 days
- Selection rate by job series
- Filters: Fiscal Year, Office, Job Series, Grade Level

## Step 5: Build the CRM Dashboard
Use:

`04_CRM_Ticket_Backlog/data/synthetic_crm_ticket_backlog.csv`

Suggested dashboard:
- Open tickets
- Average days open
- SLA met vs missed
- Ticket volume by category
- Aging open ticket table

## Step 6: Add Projects to LinkedIn
Add these to your LinkedIn Featured section:
1. Main GitHub portfolio
2. Army Housing dashboard link
3. Data Quality project
4. HR dashboard
5. CRM dashboard
6. Executive briefing PDF or summary

## Step 7: Add Portfolio Link to Resume
Use this near the top of your resume:

`Portfolio: https://github.com/YOURUSERNAME/anthony-mays-data-portfolio`

or, if using GitHub Pages:

`Portfolio: https://YOURUSERNAME.github.io/anthony-mays-data-portfolio`
